### 1. 页面一：仪表盘 / 首页 (Dashboard View)

> 对应原文档：模块 A (数据源) + 模块 B1 (暂存区展示)

这是用户进入后的第一屏，主要任务是**“筛选”**和**“分流”**。

*   **左侧栏 (Sidebar - Global)**：
    *   **上部**：All Feeds, Starred, 以及用户创建的 Topic Feed（如 `Arxiv: AI`）。
    *   **下部（核心）**：**暂存区 (Staging Area)**。这里常驻显示已 Pin 的文章。
*   **主区域 (Main List)**：
    *   文章列表（RSS 文章 + 本地 PDF）。
    *   **操作**：每行都有一个显眼的 **“Pin / 加入暂存”** 按钮。
    *   **展示**：标题、来源图标、AI 自动打的标签（`#LLM`）。
    *   **手动导入入口**：右上角“+ Import”按钮（支持粘贴 URL / 拖拽 PDF）。

---

### 2. 页面二：研读工作台 (Reader Workbench View)

> 对应原文档：模块 B (阅读与交互) + 模块 C (AI 检索)
> **这是最核心的页面，你描述的“场景”发生在这里。**

布局为 **三栏式**（或左侧栏可复用首页的左侧栏）：

*   **左侧栏 (Navigation & Staging)**：
    *   **顶部切换**：[ 目录 (TOC) ] / [ 暂存区 (Staging) ]。
    *   **暂存区状态**：用户能看到当前篮子里已经有 3 篇文章了，底部有一个高亮的 **“⚡ Generate Takeaway”** 按钮。
*   **中间栏 (Reader)**：
    *   文章正文（Markdown/PDF 渲染）。
    *   **高亮交互**：划词 -> 弹出菜单 -> 点击 "Ask AI" 或 "Add to Note"。
*   **右侧栏 (AI Assistant)**：
    *   **RAG Chat**：对话框。你问“有相关文章推荐吗？”，AI 回复并给出链接。
    *   **关键交互**：点击 AI 推荐的链接 -> **不跳转页面，而是直接将该文章加入“左侧暂存区”**（或者在新标签页打开）。

---

### 3. 页面三：笔记生成结果页 (Takeaway View)

> 对应原文档：模块 C (智能产出)

这是一个 **模态窗口 (Modal)** 或者一个 **独立的笔记详情页**。当用户在左侧暂存区点击“生成”后出现。

*   **头部**：笔记编号 `#00001`，生成时间，来源文章列表（可点击回溯）。
*   **内容区**：Markdown 编辑器。
    *   **AI 生成部分**：结构化的综述（核心概念、对比分析、结论）。
    *   **用户笔记部分**：自动聚合了用户在这些文章里的所有批注。
*   **底部操作栏**：[ 复制 Markdown ] / [ 保存到本地库 ] / [ 删除 ]。

---



左侧栏下方的 **“暂存区 (Staging)”** 是一个**全局组件**。无论用户在首页列表，还是在阅读某篇文章，这个暂存区都应该一直在（或者容易呼出），方便用户随时“扔”文章进去，最后统一“结算（生成笔记）”。
